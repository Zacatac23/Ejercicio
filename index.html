API de Incidentes

Documentación completa para gestionar incidentes

Endpoints disponibles

v1.0

Esta API permite gestionar incidentes a través de operaciones CRUD. Utiliza formato JSON para el intercambio de datos.

GET /incidentes

Obtener todos los incidentes

Devuelve una lista de todos los incidentes disponibles en el sistema.

Respuesta
Parámetros

[
  {
    "id": 1,
    "name": "Incidente 1",
    "reporte": "Descripción del incidente",
    "status": "Abierto",
    "fecha": "2024-04-03"
  },
  {
    "id": 2,
    "name": "Incidente 2",
    "reporte": "Otro incidente reportado",
    "status": "En progreso",
    "fecha": "2024-04-04"
  }
]
200 OK

Parámetros de consulta opcionales

Nombre Tipo Descripción
status string Filtrar por estado (Abierto, En progreso, Cerrado)
fecha_desde string (YYYY-MM-DD) Filtrar desde una fecha específica
POST /incidentes

Crear un nuevo incidente

Crea un nuevo registro de incidente en el sistema.

Solicitud
Respuesta

Body (JSON)


{
  "name": "Incidente 2",
  "reporte": "Otro reporte",
  "status": "En progreso",
  "fecha": "2024-04-04"
}

Campos requeridos

Campo Tipo Descripción
name* string Nombre identificativo del incidente
reporte* string Descripción detallada del incidente
status string Estado del incidente (Abierto, En progreso, Cerrado)
fecha string (YYYY-MM-DD) Fecha de registro del incidente 

{
  "message": "Incidente creado",
  "id": 3
}
201 Created
